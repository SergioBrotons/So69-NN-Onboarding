## SECCIÓN 3 — EXPLICACIÓN NUCLEAR

### 3.1 Protocolo de Respuesta

| Fase | Acción |
|------|--------|
| Detección | Identificar el problema |
| Evaluación | Determinar severidad |
| Contención | Detener el impacto |
| Resolución | Corregir el problema |
| Post-mortem | Documentar y aprender |

### 3.2 Severidad

| Nivel | Descripción | SLA |
|-------|-------------|-----|
| P1 | Sistema down | 1 hora |
| P2 | Degradación significativa | 4 horas |
| P3 | Problema menor | 24 horas |

## SECCIÓN 4 — ESCENARIO EJECUTIVO

### Situación de Presión

Un cliente llama: "El sistema no está funcionando. No hay datos desde esta mañana."

### Respuesta Correcto

"Iniciemos diagnóstico: verificar estado de sensores, revisar conectividad, comprobar procesamiento de datos. Mientras tanto, necesito información: ¿qué cambió hoy? ¿Hay algún evento en la ubicación?"
